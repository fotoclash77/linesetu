Build an **Admin Panel (Web App)** for LINESETU and connect it in real-time with:

- Patient Mobile App
- Doctor Mobile App

Use:
- React (Web) + Tailwind CSS
- Firebase Authentication
- Cloud Firestore (real-time listeners)

⚠️ This is a REAL FEATURE with correct business logic.

---

# 🎯 CORE REQUIREMENT

Admin must be able to:

- View all doctors
- Approve doctors
- Hide doctors (only from patient app)
- Delete doctors (completely remove account)

All actions must reflect **instantly in real-time** across apps.

---

# 🧱 FIRESTORE STRUCTURE

Collection: users/{userId}

Fields:

- name
- phone
- role ("doctor" | "patient" | "admin")
- isApproved (true | false)
- isActive (true | false)   // controls visibility in patient app
- isDeleted (true | false)
- createdAt

---

# 🔁 LOGIC RULES

## 1. APPROVE DOCTOR

Action: "Approve"

- isApproved = true

👉 Doctor becomes visible in Patient App (if active)

---

## 2. HIDE DOCTOR (IMPORTANT)

Action: "Hide"

- isActive = false

👉 This means:
- Doctor will NOT be visible in Patient App
- Doctor will NOT appear in:
  - doctor listing
  - search
  - booking

👉 BUT:
- Doctor can still log in to Doctor App
- Doctor account is NOT deleted

---

## 3. UNHIDE DOCTOR

- isActive = true

👉 Doctor becomes visible again in Patient App

---

## 4. DELETE DOCTOR (CRITICAL)

Action: "Delete"

This must:

1. Set:
   - isDeleted = true

2. Remove doctor access:
   - Disable login OR delete Firebase Auth user

3. Effect:

👉 Doctor must be:
- Completely removed from Patient App
- Completely removed from Doctor App
- Cannot log in anymore
- Treated as fully deleted account

---

# 📱 PATIENT APP QUERY LOGIC

Only show doctors where:

- role == "doctor"
- isApproved == true
- isActive == true
- isDeleted == false

---

# 📱 DOCTOR APP LOGIC

On login:

IF isDeleted == true:
👉 Block access (logout immediately)

---

# 📡 REAL-TIME SYNC
