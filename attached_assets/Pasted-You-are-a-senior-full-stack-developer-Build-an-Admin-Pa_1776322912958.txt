# 🎯 CORE REQUIREMENT

Create an Admin Panel where admin can:

👉 View ALL doctors (existing + new)  
👉 Approve / Hide / Delete doctors  
👉 Changes must reflect **instantly in real-time** across:
- Patient App
- Doctor App

---

# 🧱 FIRESTORE STRUCTURE

Collection: users/{userId}

Fields:

- name
- phone
- role ("doctor" | "patient" | "admin")
- isApproved (true | false)
- isActive (true | false)
- isDeleted (true | false)
- createdAt

---

# 🔁 LOGIC RULES

## 1. APPROVE DOCTOR

When admin clicks "Approve":

- isApproved = true
- Doctor becomes visible in Patient App

---

## 2. HIDE DOCTOR

When admin clicks "Hide":

- isActive = false

👉 Doctor must be hidden from:
- Patient app doctor listing
- Search results
- Booking flow

👉 Applies to:
- All users (new + old)

---

## 3. DELETE DOCTOR

When admin clicks "Delete":

- isDeleted = true

👉 Doctor must:
- Not appear anywhere in Patient App
- Not appear in Doctor App login (optional restriction)
- Be fully removed from listings

---

# 📱 PATIENT APP INTEGRATION

Modify patient app queries:

ONLY show doctors where:

- role == "doctor"
- isApproved == true
- isActive == true
- isDeleted == false

👉 This ensures real-time control from admin

---

# 📡 REAL-TIME SYNC

Use Firestore real-time listeners:
