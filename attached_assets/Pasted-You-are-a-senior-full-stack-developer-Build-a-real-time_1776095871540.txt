# 📡 REAL-TIME QUEUE LOGIC

Use Firestore listeners:

Query:

WHERE clinicId == X
AND shift == selectedShift
ORDER BY tokenNumber ASC

# 🧩 QUEUE SEGMENTATION (CRITICAL)

From same query, divide into:

1. IN CONSULTATION:
   status == "in_consultation"

2. UP NEXT:
   status == "up_next"

3. WAITING:
   status == "waiting"

4. SKIPPED:
   status == "skipped"

5. COMPLETED:
   status == "completed"

# ⚡ ACTION LOGIC

## 1. SEND NEXT (from waiting → up_next)

* Find lowest tokenNumber with status = waiting
* Update:
  status = "up_next"

---

## 2. CALL NEXT (important)

Flow:

* Move current "up_next" → "in_consultation"
* Move previous "in_consultation" → "completed"

---

## 3. SKIP

* status → "skipped"

---

## 4. CONSULTED & CALL NEXT

* current → completed
* next waiting → up_next
* auto promote up_next → in_consultation
